# CLAUDE.md

## プロジェクト概要

AWS RDS for Oracle のリザーブドインスタンス（RI）について、
為替レートをモンテカルロ法でランダムシミュレーションし、
4シナリオの年間コストを複数パターンで比較するWebアプリ。

---

- **フレームワーク**：Vue 3 + TypeScript + Vite
- **スタイル**：Tailwind CSS
- **グラフ**：Chart.js + vue-chartjs
- **バックエンド**：なし（フロントエンドのみ）
- **データ永続化**：なし

## 4シナリオ（固定）

| シナリオ | 割引率 | 支払い方法 |
|---------|--------|-----------|
| オンデマンド | なし | 毎月その月の為替レートで円換算 |
| RI・前払いなし | 28%割安 | 毎月その月の為替レートで円換算 |
| RI・一部前払い | 30%割安 | 前払い分は1ヶ月目のレートで円換算。月額分は毎月のレートで円換算 |
| RI・全額前払い | 32%割安 | 年間一括を1ヶ月目のレートで円換算。2〜12ヶ月目は0円 |

### RI・一部前払いについての補足

RI・一部前払いは割引率（30%）がRI・全額前払い（32%）より低く、
かつ月額部分に為替変動リスクが残るため、
年間支払総額の比較では全額前払いに勝るシナリオは発生しない。

一部前払いは「手元資金を温存しながら割引を受けたい」場合のキャッシュフロー上の選択肢であり、
純粋なコスト最小化を目的とする場合は全額前払いが常に有利となる。
これはコードのバグではなくAWSの料金設計の仕様である。

### 計算式

```
onDemandMonthly_USD = hourlyRate × hoursPerMonth
riNoUpfront_USD     = onDemandMonthly_USD × 0.72
riFullUpfront_USD   = onDemandMonthly_USD × 12 × 0.68
```

一部前払いの前払い額・月額はユーザー入力値をそのまま使用する。

## 為替モデル（モンテカルロ）

### 入力パラメータ

| パラメータ | 説明 | デフォルト値 |
|-----------|------|------------|
| 現在レート | シミュレーション開始値（1ヶ月目） | 160円 |
| 想定下限 | これ以下になりにくい（超えにくいがあり得る） | 150円 |
| 想定上限 | これ以上になりにくい（超えにくいがあり得る） | 170円 |
| 介入ライン | これを超えると円高バイアスが強まる | 160円 |
| 月次変動幅 | 前月からの最大変動幅（±） | 3円 |
| 通常時円高確率 | 通常時に円高方向に動く確率 | 60% |
| 介入時円高確率 | 介入ライン超え時に円高方向に動く確率 | 80% |
| シミュレーション回数 | 10〜100回 | 100回 |

### 為替生成ロジック

1ヶ月ごとに以下のルールで次月レートを生成する：

```
1. 現在レートが介入ラインを超えているか判定
2. 超えている場合は介入時円高確率、そうでなければ通常時円高確率を使用
3. 確率に従って変動方向（円高 or 円安）を決定
4. 変動幅（0〜月次変動幅）をランダムで決定
5. 次月レート = 現在レート ± 変動幅
6. 想定下限・上限に近づくほど逆方向に動く確率を高める（ソフトバリア）
```

## シミュレーション結果の表示

### 1. 最安シナリオの集計
各シミュレーション回で最安だったシナリオを集計し、シナリオごとの最安回数と確率を表示。
例：RI全額前払い 45回 / 45%

### 2. 年間支払額の分布
シナリオごとの最小・最大・平均を表示。

### 3. 全パターンの為替推移グラフ
全シミュレーション回の12ヶ月為替推移を折れ線グラフで重ねて表示。

### 4. パターンごとの明細テーブル
アコーディオンでデフォルト折りたたみ。
列構成：パターン番号 ｜ 1〜12月の為替レート ｜ 各シナリオの年間支払額（円） ｜ 最安シナリオ

## ファイル構成

```
src/
  App.vue
  types.ts
  constants.ts
  utils/
    calculator.ts           // 4シナリオの年間支払額計算（純粋関数）
    montecarlo.ts           // 為替レート生成ロジック（純粋関数）
  composables/
    useSimulator.ts         // 入力値・シミュレーション結果の管理
  components/
    ScenarioSettings.vue    // シナリオ入力値（時間単価・稼働時間・一部前払い額）
    MontecarloSettings.vue  // 為替モデルのパラメータ設定
    ResultSummary.vue       // 最安シナリオの回数・確率
    ResultDistribution.vue  // 年間支払額の分布（最小・最大・平均）
    ResultFxChart.vue       // 全パターンの為替推移グラフ
    ResultDetailTable.vue   // パターンごとの明細テーブル（アコーディオン）
```

## UI要件

- シミュレーション実行ボタンを押したタイミングで計算する（リアルタイム更新ではない）
- 数値表示：円は小数点なし・3桁カンマ、ドルは小数点2桁
- レスポンシブ対応（スマホ〜PC）
- バリデーションエラーはフィールドを赤枠表示
- ResultDetailTableはデフォルトで折りたたみ表示
